Dynamic Programming

Climbing Stairs

题目描述
给定 n 节台阶，每次可以走一步或走两步，求一共有多少种方式可以走完这些台阶。
输入输出样例
输入是一个数字，表示台阶数量；输出是爬台阶的总方式。

题解

斐波那契，设 d[i] 表示走到第 i 阶的方法数。因为每次可以走一步或者两步，所以第 i 阶可以从第 i -1 阶或 i - 2 阶到达。故走到第 i 阶的方法数等于走到第 i - 1 阶的方法数加上第 i - 2 阶的方法数，从而得到状态转移方程 d[i] = d[i-1] + d[i-2]

动态规划的核心就是求状态转移方程并且避免重复运算

House Robber

题目描述
假如你是一个劫匪，并且决定抢劫一条街上的房子，每个房子内的钱财数量各不相同。如果
你抢了两栋相邻的房子，则会触发警报机关。求在不触发机关的情况下最多可以抢劫多少钱。
输入输出样例
输入是一个一维数组，表示每个房子的钱财数量；输出是劫匪可以最多抢劫的钱财数量。

题解
定义一个数组 dp，dp[i] 表示抢劫到第 i 个房子时，可以抢劫的最大数量。我们考虑 dp[i]，
此时可以抢劫的最大数量有两种可能，一种是我们选择不抢劫这个房子，此时累计的金额即为
dp[i-1]；另一种是我们选择抢劫这个房子，但也由此可知我们不
能够抢劫第 i-1 个房子（否则会触发警报机关），所以此时累计的最大金额只能是 dp[i-2] + nums[i] 。因此本题的状态转移方程为 dp[i] = max(dp[i-1], nums[i] + dp[i-2])

Arithmetic Slices

题目描述

给定一个数组，求这个数组中等差的子数组一共有多少个

输入输出样例

输入是一个一维数组，输出是满足等差条件的连续子数组个数

Input: nums = [1,2,3,4,7,7,8,9]

Output: 4

在这个样例中，等差数列有 [1,2,3], [2,3,4], [1,2,3,4], [7,8,9]

题解

设 dp[i] 表示以原数组第 i 个元素为结尾的等差数组个数，状态转移方程为 dp[i] = dp[i-1] + 1 （需注意，dp[i] 可为 0）

则总等差数组个数为 dp[i]

建一个长度为 3 的窗口 triple ，如满足 tripe[3] - triple[2] = triple[2] - triple[1] （不使用加法以防止整型溢出；不用位运算因数组元素可能为负），则满足条件，记录即可

空间复杂度优化：用一个变量去储存上一个 dp ，用一个变量去储存 sum

给定一个 m x n 大小的非负整数矩阵，求从左上角开始到右下角结束的、经过数字和最小的路径；。每次只能向右移动一格或向下移动一格。

输入输出样例

输入是一个二维数组，输出是最优路径的数字和

说明：在这个样例中，最短路径为 1->3->1->1->1

题解

设 dp[i][j] 表示从左上角开始到 (i,j) 位置的最优路径长，则状态转移方程为：

dp[i][j] = min(dp[i-1][j], dp[i][j-1]) + grid[i][j] （grid 表示给定的矩阵）

注意边界讨论即可

输入输出样例

输入是一个二维数组，输出一个 32 位整型

输入输出样例

输入是一个以一维数组，输出是一个正整数

解法1

将 dp[i] 和 max 都初始化为 1，表示当 nums[i] 之前的所有元素都不小于 nums[i] 时，dp[i] 为 1（以它为结尾的最长子序列只有它自身）

因为要求所有子序列中最长的子序列的长度，所以用变量 max 去记录最大的 dp[i]

此解法时间复杂度为 O(n2)

解法2

此题可用贪心算法，每次都储存递增子序列的末位最小值。因为子序列递增，所以当子序列长度一定时，末位越小，后续能容纳的序列也就越多。最终也可得到最长子序列长度，时间复杂度为 O(logn), 具体为实现如下：

新建数组 d，设 d[i] 表示长度为 i + 1 的所有递增子序列中的末位的最小值。比如，d = [2, 5] 中，d[0] = 2 表示在所有长为 1 的递增子序列中，递增子序列末位的最小值是 2 ，d[1] 表示所有长为 2 的递增子序列中，递增子序列末位的最小值是 5 。符合这个条件的 nums 可以是 [3,7,2,5] ，因为长为 1 的递增子序列分别为 [3] , [7] , [2] , [5] , 其中末位最小为 2 ；长为 2 的递增子序列有 [3,7] , [2,5] , [3,5] ，其中末位最小为 5

因为 d[i] 表示长度为 i + 1 的所有递增子序列中的末位的最小值，所以数组 d 的长度即为子序列长度的最大值。要维护此数组 d ，需要对 nums 进行遍历，并采用如下策略（利用 d 的递增性）：

  • 如果 d 为空，则 d[0] = num
  • 如果 num 比 d[0] 要小，则 d[0] = num

  • 如果 num 比 d 的末位元素要大，则 d.push(num) ，表示得到了一个更长的子序列，其末位为 num

  • 如果 num 介于 d 的两个元素之间，则将后一个元素替换成 num

  • 否则，不进行操作，并读取下一个 num

优化

因为 d 是严格增序的，所以查找 num < d[i] 时不必线性遍历，可以用二分查找

这里利用 rust 的 binary_search(num).unwrap_or_else(|x| x ) ，可以产生如下效果：

  • 如果数组中每一位都比 num 小（换言之，num 大于数组的末位），则返回数组末位后一位的 index
  • 如果数组中有 num ，则返回其 index
  • 如果数组中每一位都比 num 大（换言之，num 小于数组第一位），则返回 index = 0
  • 如果 num 介于数组某两位之间，则返回后一位（较大的那一位）的 index
